Remarks

This operation executes immediately against the database, rather than being deferred until SaveChangesAsync(CancellationToken) is called. It also does not interact with the EF change tracker in any way: entity instances which happen to be tracked when this operation is invoked aren't taken into account, and aren't updated to reflect the changes.

See Executing bulk operations with EF Core for more information and examples.
